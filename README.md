# 音乐排练室预约墙

一个现代化的音乐排练室预约系统前端，支持乐队成员查看排练室和乐器设备，场地管理员维护时段、押金和故障设备。

## 🎯 入口地址
- **开发环境**: http://localhost:3000
- **Docker部署**: http://localhost:8080

## 👤 测试账号

默认内置用户（本地存储模拟）

| 角色 | 用户名 | 说明 |
|------|--------|------|
| 普通用户 | 张三 | 默认登录，押金已缴纳 |
| 普通用户 | 李四 | 默认登录，押金未缴纳 |

> 💡 点击右上角「切换到管理员模式」可切换至管理员视图，无需额外登录

## ✨ 功能特性

### 乐队成员功能
- **折叠分组展示**：支持按类型、容量、价格三种分组方式，可展开/折叠每组房间
- **移动端适配**：折叠分组操作在移动宽度下完美适配
- **房间筛选**：按容纳人数、价格范围、设备需求筛选排练室
- **房间卡片**：展示排练室图片、容纳人数、面积、价格、设备标签
- **设备标签**：显示所有设备，故障设备标红并展示故障描述
- **时段选择**：可视化时段选择，黄金时段特别标注
- **预约表单**：填写乐队信息、联系人信息
- **预约草稿**：保存未完成的预约，随时加载继续
- **本地历史**：查看所有历史预约记录
- **押金提示**：触发押金限制时给出清晰原因说明

### 场地管理员功能
- **时段管理**：配置普通时段和黄金时段
- **押金管理**：管理用户押金缴纳状态
- **设备管理**：标记设备故障，填写故障描述
- **故障提示**：故障设备自动在房间卡片显示警告

### 核心业务规则
- ❗ **押金未缴不能预约黄金时段**（18:00-22:00），触发时给出详细原因
- ⚠️ **故障设备在房间卡片提示
- 💾 数据全部存储在浏览器本地存储

## 🐳 Docker 容器启动

### 构建镜像
```bash
docker build -t rehearsal-booking .
```

### 启动容器
```bash
docker run -p 8080:80 --name rehearsal-booking rehearsal-booking
```

### 访问应用
打开浏览器访问 http://localhost:8080

### 使用 Docker Compose（可选）
创建 `docker-compose.yml`:
```yaml
version: '3'
services:
  web:
    build: .
    ports:
      - "8080:80"
    restart: unless-stopped
```

启动：
```bash
docker-compose up -d
```

## 📖 使用说明

### 验证折叠分组功能
1. 打开应用，进入用户模式
2. 在房间列表上方找到「分组方式」选择器
3. 尝试切换「按类型」「按容量」「按价格」三种分组方式
4. ✅ **验证结果**：房间按选择的维度自动分组，每组可点击标题可展开/折叠
5. 缩小浏览器窗口至移动宽度（<768px）
6. ✅ **验证结果**：分组选择器和折叠按钮自动换行，折叠分组操作正常

### 验证押金限制功能
1. 打开应用，默认用户为「张三」，押金状态为「已缴纳」
2. 点击右上角「切换到管理员模式」
3. 进入「押金管理」标签页，将张三的押金标记为「未缴」
4. 切换回用户模式，选择任意排练室
5. 尝试选择 18:00-22:00 之间的黄金时段
6. ✅ **验证结果**：黄金时段按钮变为禁用状态，显示「押金未缴」提示，鼠标悬停显示原因
7. 下方出现黄色警告框，详细说明押金未缴的原因和解决方法
8. 填写表单后，「确认预约」按钮保持禁用，无法提交预约

### 验证故障设备提示
1. 切换到管理员模式
2. 进入「设备管理」标签页
3. 选择一个排练室，将某个设备标记为故障
4. 填写故障描述（如「踩锤弹簧损坏」）
5. 切换回用户模式
6. ✅ **验证结果**：对应房间卡片出现「⚠️ 设备故障」红色徽章，下方展示故障设备列表

### 预约草稿功能
1. 选择房间和时段，填写部分表单信息
2. 点击「保存草稿」
3. 刷新页面或重新进入
4. 左侧边栏「预约草稿」区域显示保存的草稿
5. 点击「加载」即可恢复之前填写的内容

## 🏗️ 技术栈

- **React 18** - UI 框架
- **TypeScript** - 类型安全
- **Vite** - 构建工具
- **CSS3** - 样式（原生 CSS，无 UI 库依赖）
- **LocalStorage** - 本地数据持久化
